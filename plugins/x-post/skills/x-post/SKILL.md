# Xポスト自動生成スキル

> **【最初に1回だけ設定】** 発信の核・ターゲット・トーン・NG・CTA を `~/.claude/x-post-config.md` に書いておくと、毎回の質問が激減します。ファイルが無ければ **Step 0.5 で Claude が対話しながら作ります**。雛形は同梱の `config.example.md` を参照。

X（Twitter）の投稿案を生成するスキル。実運用ツール XToolsPro2 / XToolsPro4 に組み込まれていた生成プロンプトを、そのままスキルとして切り出したもの。**外部APIキーもPythonも不要**で、文章は Claude が直接書く。

## 使用方法

```
/x-post <テーマ または 参考URL> [モード] [件数]
```

- **引数なし**でもOK（対話で聞く）
- モード指定語: `深掘り` / `A` / `量産` / `B`
- 件数指定: `10件` `50件` など

例:
- `/x-post 副業の始め方`
- `/x-post https://note.com/xxx/n/xxxx 深掘り`
- `/x-post 時間管理 量産 50件`

## 定数

- **設定ファイル**: `~/.claude/x-post-config.md`
- **設定の雛形**: `${CLAUDE_PLUGIN_ROOT}/config.example.md`
- **プロンプト置き場**: `${CLAUDE_PLUGIN_ROOT}/skills/x-post/prompts/`
  - `prompt-深掘り型.md` — モードA本体
  - `prompt-量産型.md` — モードB本体
  - `prompt-AI臭を消す8手法.md` — ❶〜❽の全文（生成・推敲の両方で使う）
  - `prompt-10パターン.md` — モードBの型カタログ
- **既定の保存先**: `./x-posts/`（設定ファイルで変更可）

> `${CLAUDE_PLUGIN_ROOT}` が解決できない環境では、Glob で `**/skills/x-post/prompts/prompt-*.md` を探してフォールバックする。

## 2モード早見表

| | **モードA 深掘り型** | **モードB 量産型** |
|---|---|---|
| 由来 | XToolsPro4 post-generator | XToolsPro2 AIモード |
| ハッシュタグ | **禁止** | **必須（2〜3個）** |
| URL | 禁止 | CTA行として可 |
| 文字数 | 280字以内 ／ 長文は400〜1,200字 | 140単位 or 280単位 |
| 件数 | 5 / 10 / 20 | 10 / 50 / 100 |
| 改行整形 | 自然な範囲で | 1〜2行ごとに空行、。！？の後で改行 |
| 各案の付帯情報 | 切り口名＋自己評価スコア（0.00〜1.00） | パターン名（10種） |
| 設計手法 | AI臭を消す8手法（A/B/C） | 10パターン＋成功事例7指針 |
| 想定用途 | 本命ポスト・ブランディング | ランダム投稿キューのストック |

## ワークフロー

### Step 0: 引数解析と設定読み込み

1. 引数を分解する:
   - `http://` / `https://` で始まる語 → **参考URL**
   - `深掘り|AI臭|丁寧|ブランディング|A` → モードA
   - `量産|大量|キュー|ハッシュタグ|B` → モードB
   - `\d+件` の数値 → 件数（5/10/20 ならA寄り、50/100 ならB寄りのヒント）
   - 残り → **テーマ**
2. Read で `~/.claude/x-post-config.md` を読む。
   - 存在すれば内容を保持し、Step 2 以降の既定値として使う
   - 存在しなければ **Step 0.5** へ

### Step 0.5: 初回セットアップ（設定ファイルが無い時だけ）

1. `${CLAUDE_PLUGIN_ROOT}/config.example.md` を Read する
2. AskUserQuestion で「今すぐ設定を作る／今回はスキップ」を確認する
3. 「作る」なら **1回の AskUserQuestion（最大4問）** で以下を聞き、残りは自由記述で補う:
   - 発信ジャンル・テーマ領域
   - ターゲット読者
   - 投稿のトーン
   - CTA（誘導先・なしも可）
4. Write で `~/.claude/x-post-config.md` に保存し、パスをユーザーに伝える
5. 「スキップ」なら設定なしのまま Step 1 へ進む（毎回聞くことになる旨を1行で伝える）

### Step 1: モード決定

Step 0 の推論でモードが確定していればスキップ。確定していなければ **AskUserQuestion を1問だけ**出す。

> モード選択は必ず単独1問にする。モードが決まるまで件数の選択肢（5/10/20 か 10/50/100 か）が確定しないため。

| 選択肢 | 説明 |
|---|---|
| A 深掘り型 | AI臭を消して1本ずつ丁寧に。ハッシュタグなし。本命ポスト向け |
| B 量産型 | 10パターンで一気にストック。ハッシュタグ付き。ランダム投稿のキュー向け |

参考URLが渡されていた場合はモードAを既定として提示する（URL要約はモードAの機能）。

## モードA：深掘り型

### Step 2A: 生成条件のヒアリング

`${CLAUDE_PLUGIN_ROOT}/skills/x-post/prompts/prompt-深掘り型.md` を Read する。

設定ファイルがあれば、7問アンケートのうち **genre / audience / tone / keywords / avoid / reference の6項目は設定から自動充填**する。
足りない項目だけを **AskUserQuestion 1回（最大4問）** で聞く:

| 質問 | 選択肢 |
|---|---|
| 生成する件数は？ | 5件 / 10件（推奨） / 20件 |
| 文字数は？ | 短文（280字以内・X通常投稿） / 長文（400〜1,200字・note/Threads兼用） |
| 文章設計パターンは？ | A 構造から組み立てる（❶❹❼） / B 主張型の記事（❷❸❺） / C 仕上げで精度を上げる（❻❽❹） / 指定なし |
| 投稿の目的は？（設定に無い時だけ） | 情報発信・教育 / 共感・ストーリー / 集客・販促 / ファン化・関係構築 |

### Step 3A: 材料収集

1. 参考URLがあれば WebFetch で本文を取得し、`## 参考にする元コンテンツ` ブロックを組み立てる
2. ユーザーが過去の反応が良かった投稿を貼っていれば、それも材料に含める
3. 設定ファイルの「参考にしたい言い回し」を反映する

### Step 4A: 生成

1. `prompt-AI臭を消す8手法.md` を Read し、選ばれたパターン（A/B/C）の3手法のエッセンスと共通ディレクティブを**生成ルールとして内在化する**（外部APIには送らない。Claude 自身が書く）
2. `prompt-深掘り型.md` のシステムプロンプト＋指示文の条件に従って、指定件数を生成する
3. **絶対ルール**:
   - ハッシュタグを付けない
   - URLを付けない
   - 短文モードは280文字以内を厳守（生成後に必ず1件ずつ文字数を数えて確認する）
   - 設計の過程（❶の結論1行、❸の対立軸など）は出力せず、最終本文にだけ反映する
4. 各案に **切り口の短い名前** と **自己評価スコア（0.00〜1.00）** を付ける

→ Step 5 へ

## モードB：量産型

### Step 2B: 生成条件のヒアリング

`${CLAUDE_PLUGIN_ROOT}/skills/x-post/prompts/prompt-量産型.md` と `prompt-10パターン.md` を Read する。

設定ファイルがあれば target / cta を自動充填。
**AskUserQuestion 1回（最大4問）** で聞く:

| 質問 | 選択肢 |
|---|---|
| 生成する件数は？ | 10件（推奨） / 50件 / 100件 |
| 文字数は？ | 140単位（通常アカウント） / 280単位（Premium） |
| 目的は？ | 認知拡大 / ファン化 |
| 感情トーンは？ | 前向き・背中を押す / 共感・寄り添い / 驚き・発見 / 冷静・分析的 |

### Step 3B: 「魂」ヒアリング

Step 2B の「目的」に応じて、**自由記述で1回だけ**聞く（原文どおり）:

- 認知拡大 → 「今回のテーマで、多くの人が『なるほど！』と驚くような、核心をついた一言や、常識を覆す事実はありますか？」
- ファン化 → 「今回のテーマについて、あなたが体験した具体的なエピソードや、特に心が大きく動いた瞬間があれば教えてください。」

**スキップ可**。未回答なら `（お任せ）` として扱う。

### Step 4B: 生成

1. `prompt-量産型.md` の生成プロンプト条件に従って生成する
2. **10件を1バッチ**として分割する。11件以上なら2バッチ目以降で「既出と重複しない」ことを自ら課す
3. 各件に10パターンのいずれかを割り当てる（11件目以降は1〜10を循環）
4. **絶対ルール**:
   - 各投稿の最後にハッシュタグを2〜3個必ず付ける（`#` で始める）
   - 本文は1〜2行ごとに空行を入れる。「。」「！」「？」の後で改行する
   - 文字数は **X の重み付け**（日本語1文字＝2、英数字1文字＝1）で数える。
     実務上の上限は **140単位＝全角140文字／280単位＝全角280文字**（ハッシュタグ込み）として扱う。
     原典 `ai_generator.py` の「日本語2文字＝1単位」という表記は逆の言い回しだが、
     指す上限（通常アカウント＝全角140文字、Premium＝全角280文字）は同じ。
   - **数字を創作しない**。実績が提供されていなければ、数字前提のパターン（1・3・6・9）は他の型に振り替えるか、数字の有無をユーザーに確認する

→ Step 5 へ

---

### Step 5: ファイル保存

保存先 = 設定ファイルの「保存フォルダ」／未設定なら `./x-posts/`（無ければ作る）。

ファイル名: `x-post_{YYYYMMDD-HHmm}_{A深掘り|B量産}_{テーマslug}.md`
（同名があれば `_2` `_3` と連番を付ける。既存ファイルは上書きしない）

**モードAの中身**

```markdown
---
mode: A 深掘り型
theme: 副業の始め方
count: 10
length: short
pattern: A
generated: 2026-07-20 14:30
---

## 1. 逆算で語る ／ score 0.82

（本文）

*138文字*

## 2. ...

---

## スコア上位3件

1. #5 常識への反証（0.88）
2. #1 逆算で語る（0.82）
3. #7 数字フック（0.79）
```

**モードBの中身**

```markdown
---
mode: B 量産型
theme: 時間管理
count: 50
char_unit: 140
purpose: 認知拡大
generated: 2026-07-20 14:30
---

## 1. 数字インパクト型（Before/After）

（改行整形済み本文）

#時間管理 #朝活

## 2. ...

---

## パターン別内訳

| パターン | 件数 |
|---|---|
| 数字インパクト型 | 5 |
| ... | ... |
```

保存後、**絶対パスとファイルサイズ**をユーザーに報告する。

### Step 6（任意）: 推敲

ユーザーが「もっと磨きたい」と言った場合のみ実行する。

1. 対象の投稿番号を確認する
2. `prompt-AI臭を消す8手法.md` の該当手法の**指示文**をそのまま適用して推敲する
   - 冒頭が弱い → ❼ 冒頭3パターン
   - 抽象的で刺さらない → ❻ 抽象度診断
   - 主張が弱い → ❸ 対立構造 ／ ❺ 想定反論
   - 記憶に残らない → ❹ 核フレーズ抽出
   - 最後まで読まれない → ❽ 読者離脱予測
3. 推敲後は元ファイルに追記する（元の案は消さない）

### Step 7（任意）: キュー投入用テキスト

ユーザーが「コピペ用が欲しい」と言った場合のみ、同名 `.md` の隣に `{同名}_queue.txt` を書き出す。
中身は**本文だけ**を `---` 単独行で区切ったもの（見出し・スコア・frontmatter を含めない）。

> ⚠️ **CSVは出力しない。** XToolsPro2 / XToolsPro4 のいずれにも「投稿本文のCSV/TSV一括インポート」機能は存在しない。
> XToolsPro2 の CSV/XLSX 一括登録は**アカウント登録専用**であり、投稿本文用ではない。
> ランダム投稿キューへの投入は、現状ツール画面での貼り付けが正しい経路。

## ガイドライン

- **数字・実績を創作しない。** 手元に無い数字は書かない。これが最大の事故源。
- **他人の投稿をそのまま複製しない。** 参考URLの本文は再構成する（コピーしない）。
- **モードのルールを混ぜない。** モードAにハッシュタグを付けない、モードBのハッシュタグを省かない。
- 文字数は生成後に必ず数え直す。超過したら短縮して再提示する。
- 設定ファイルに書かれた **NGワード・避けたい話題**は絶対に破らない。
- 生成した投稿は下書きである。投稿前にユーザー自身の目で確認してもらう旨を、初回だけ伝える。
