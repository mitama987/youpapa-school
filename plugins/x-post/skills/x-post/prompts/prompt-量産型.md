# モードB「量産型」プロンプト原文

> 出典: XToolsPro2 POST（ランダム投稿 → AIモード）
> - `ai_generator.py`（システムプロンプト・生成プロンプト）
> - `static/js/ai-mode-manager.js`（「魂」ヒアリングの質問文）
>
> **ハッシュタグ2〜3個は必須**。ランダム投稿のキューに積むためのストック生成用。

## システムプロンプト

```
指定されたポスト数のみを生成してください。
余計なコメントや説明は不要です。投稿文のみを出力してください。

出力フォーマット：
1. パターン名
投稿本文
#ハッシュタグ

2. パターン名
投稿本文
#ハッシュタグ
...
```

## ヒアリング項目（6問）

| 項目 | 内容 | 選択肢 |
|---|---|---|
| theme | テーマ | 自由記述 |
| purpose | 目的 | 認知拡大／ファン化 |
| target | ターゲット | 自由記述 |
| character_count | 文字数 | 140単位／280単位 |
| emotion | 感情トーン | 自由記述（例: 前向き、共感、驚き） |
| cta | CTA | 自由記述（なしも可） |

### 「魂」（核心的なメッセージ）のヒアリング

目的の選択によって聞き方を変える（原文）:

- **認知拡大** を選んだ場合:
  > 今回のテーマで、多くの人が「なるほど！」と驚くような、核心をついた一言や、常識を覆す事実はありますか？

- **ファン化** を選んだ場合:
  > 今回のテーマについて、あなたが体験した具体的なエピソードや、特に心が大きく動いた瞬間があれば教えてください。

回答は任意。未回答なら `（お任せ）` として扱う。

## 文字数のカウント規則（原文）

```
【文字数制限】
- 制限: {char_limit}単位以内（X/Twitter準拠）
- 日本語2文字=1単位、英数字1文字=1単位
- 140単位の場合：本文100単位程度 + ハッシュタグ2個（#含めて20単位程度）
- 280単位の場合：本文220単位程度 + ハッシュタグ3個（#含めて40単位程度）
※ハッシュタグは必ず「#」記号で始めてください
```

> 注: 上は原典の逐語。「日本語2文字=1単位」は原典の逆言い回しで、Xの実仕様は日本語1文字＝重み2。
> 本スキルでは全角文字数に統一し、**通常＝全角140文字以内／Premium＝全角280文字以内（本文＋ハッシュタグ込み）** として扱う。
> 数え方は SKILL.md の「文字数の数え方（X共通）」に従うこと。

## 成功事例に基づく作成指針（原文・7項目）

```
【成功事例に基づく作成指針】
1. **必ず具体的な数字を含める**（改善率、期間、実績など）
   例：「表示率15％→33％」「たった1週間で」「反応率3倍」
2. **Before/Afterを明確に対比**させる
3. **読者の痛みや課題に共感**してから解決策を提示
4. **感情に訴える表現**（驚き、発見、安心、期待）
5. **視覚的な装飾**（→ などの記号や絵文字）を効果的に使用
6. **最初の1行でフック**を作る（問いかけ、衝撃的な事実、共感）
7. **実現可能性**を示す（期間、簡単さ、確実性）
```

> 注: 原典の4・5番は絵文字が含まれるが、ソースファイル上で文字化けしているため意味を保って整理した。絵文字の使用可否はユーザー設定（トーン・NG）に従う。

## 10パターン（型カタログ）

`prompt-10パターン.md` を参照。指示文としては以下の原文を使う。

```
以下の10パターンで、**深みのある魅力的な投稿**を生成してください。
各投稿は必ず最後にハッシュタグを2-3個含めてください。
各パターンで内容が被らないようにし、会話内容に沿って具体的に書いてください。

1. 数字インパクト型（Before/After）
2. 問題提起→解決型
3. 実績ストーリー型
4. 驚きの発見型
5. 共感から希望へ型
6. 緊急性×実績型
7. 比較で納得型
8. 体験者の声型
9. How-to実績型
10. ビフォーアフター感動型

重要：各投稿は実際の数字と感情を込めて生成してください。読者の心を動かす内容にしてください。
```

## バッチ生成について

原典は 10件を1バッチとして分割生成する（`batch_size = 10`）。11件以上のときは10件ずつに分け、
各プロンプト冒頭に「バッチ注記」として `（N回中M回目）` の**位置表示のみ**を付ける
（`ai_generator.py` の `batch_note = f"（{total_batches}回中{batch_index + 1}回目）"`）。

※原典では各バッチが独立したLLM呼び出しで、前バッチの出力を次バッチへ渡さないため、
原典に **《既出投稿との内容重複を回避する》ロジックは存在しない**。
本スキルでは Claude が全件を1つの文脈で生成できるので、SKILL.md Step 4B のとおり
「11件以上なら2バッチ目以降で既出と重複しない」ことを **スキル独自の上乗せ（原典に無い追加仕様）** として課す。
